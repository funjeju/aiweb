# 1. Live Feed System Overview (실시간 운영 피드 시스템)

본 기능의 목적은 실시간 관광 데이터 수집이 아니다. 핵심은 **웹사이트를 살아있는 상태로 유지하는 것**이다.

기존 소상공인 홈페이지는 대부분 제작 이후 방치된다. 메뉴가 바뀌어도 업데이트되지 않고, 분위기나 현재 운영 상태가 반영되지 않아 결국 “죽은 홈페이지”가 된다.

본 시스템은 이러한 문제를 해결하기 위해 **운영 중 자연스럽게 업데이트되는 웹사이트 구조**를 지향한다.

즉,

> 만드는 홈페이지가 아니라
> 살아 움직이는 홈페이지

를 목표로 한다.

Live Feed는 웹빌더의 핵심 기능이 아니라 **웹사이트 운영 활성화 시스템(Operational Layer)** 으로 동작한다.

---

# 2. Core Philosophy (핵심 철학)

운영은 어려우면 안 된다.

사장님은 콘텐츠 마케터가 아니다. 대행자 역시 운영 피드를 만들기 위해 별도 툴을 쓰고 싶어하지 않는다.

따라서 본 시스템은 다음 원칙을 따른다.

> 사진 한 장 올리면 끝

즉, 사용자가 매장 사진을 업로드하면 AI가 이를 분석하여 웹사이트에 자연스럽게 반영해야 한다.

예:

신메뉴 사진 업로드

→ 오늘의 메뉴 카드 생성

→ 웹사이트 Featured Section 반영

→ SNS 카드 생성 가능

카페 전경 사진 업로드

→ Today Mood 업데이트

→ Hero 이미지 추천 교체

→ 분위기 카피 자동 생성

핵심은:

> 운영의 마찰(Friction)을 제거한다

이다.

---

# 3. Why Live Feed Matters (왜 필요한가)

웹사이트는 제작보다 유지가 더 어렵다.

소상공인의 실제 문제는 다음과 같다.

* 홈페이지를 안 고침
* 메뉴 업데이트 안 함
* 공지 안 올림
* 계절/이벤트 반영 안 함

결국 사이트가 오래된 느낌을 주게 된다.

Live Feed는 이러한 문제를 해결하기 위한 구조다.

즉,

> 웹사이트 운영을 콘텐츠 업로드 수준으로 단순화

하는 것이 목적이다.

---

# 4. What is Live Feed? (Live Feed 정의)

Live Feed는 SNS 피드가 아니다.

정확히는:

> 웹사이트의 “지금 상태”를 보여주는 운영 레이어

다.

예시:

카페

* 오늘의 디저트
* 오늘 노을 사진
* 오늘 분위기

식당

* 오늘 추천 메뉴
* 신메뉴 출시
* 오늘 재료

펜션

* 오늘 객실 뷰
* 오늘 날씨 분위기
* 공지사항

미용실

* 오늘 스타일 샷
* 신규 스타일 소개
* 예약 유도 콘텐츠

핵심은 업종별 운영 흐름에 맞는 자동 피드다.

---

# 5. EXIF-Based Context Detection

사진 업로드 시 EXIF 메타데이터를 활용할 수 있다.

단, EXIF는 주인공이 아니다.

역할은:

> 보조 맥락(Context Hint)

이다.

활용 예시:

촬영 시간

→ 오전/노을/야간 분위기 판단

위치 정보(가능 시)

→ 매장 촬영 여부 추정

연속 업로드 패턴

→ 오늘 운영 콘텐츠 후보 판단

예:

오후 7시 오션뷰 사진

↓

“오늘 노을이 정말 예쁜 저녁입니다” 카피 제안

중요한 점은:

> EXIF에 의존하지 않는다

이다.

메타데이터가 없어도 사진 기반 분석으로 동작해야 한다.

---

# 6. AI Feed Structuring Pipeline

사진 업로드 시 AI는 다음을 수행한다.

1. 사진 분석
2. 업종 맥락 이해
3. 콘텐츠 유형 분류
4. 카피 생성(Optional)
5. 사이트 반영 추천

예시:

입력:

케이크 사진

↓

판단:

신메뉴 가능성

↓

자동 제안:

“오늘의 디저트” 카드 생성

↓

웹사이트 Featured Block 추천

또는:

오션뷰 사진

↓

분위기 콘텐츠

↓

Today Mood Section 반영

즉,

> AI가 운영 콘텐츠를 구조화

한다.

---

# 7. Website Integration (웹사이트 연동)

Live Feed는 별도 페이지가 아니다.

기존 웹사이트 블록과 자연스럽게 연결된다.

예:

Hero Block

→ 오늘 사진 추천

Featured Block

→ 신메뉴 카드 삽입

Gallery

→ 최신 업로드 자동 반영

Announcement Block

→ 이벤트 카드 생성

즉,

> 기존 웹사이트가 점점 살아난다

는 느낌을 줘야 한다.

---

# 8. Suggested UX (핵심 UX)

사용자 경험은 극도로 단순해야 한다.

예시 흐름:

사진 업로드

↓

AI 분석

↓

자동 제안

“오늘의 메뉴로 추가할까요?”

“이벤트 카드 생성할까요?”

↓

원클릭 적용

핵심은:

> 수정은 최소화, 승인만 하게 만든다

이다.

---

# 9. Agency Workflow

대행자 관점에서도 운영 효율이 중요하다.

예:

매장 사진 전달받음

↓

업로드

↓

자동 피드 생성

↓

승인

↓

즉시 반영

즉,

> 운영 대행 생산성 향상

에도 기여해야 한다.

---

# 10. Future Layer: DaaS Possibility

향후 확장 가능성으로 Live Feed 데이터는 로컬 운영 데이터를 축적할 수 있다.

예:

* 계절별 메뉴 패턴
* 시간대별 분위기 콘텐츠
* 업종별 운영 트렌드

그러나 이는 어디까지나 **후면 레이어(Backend Possibility)** 다.

제품 관점 우선순위는 다음과 같다.

> 웹빌더 운영 활성화
> → 살아있는 웹사이트
> → 이후 데이터 확장

즉, DaaS는 사용자에게 보이는 핵심 가치가 아니다.

---

# 11. Final Principle

본 기능의 핵심은 다음 한 문장으로 정리된다.

> 죽어있는 홈페이지가 아니라
> 운영되는 홈페이지를 만든다

그리고 사용자는 복잡한 CMS를 쓰는 것이 아니라,

> 사진 한 장 올리고 승인만 한다

는 경험을 가져야 한다.
