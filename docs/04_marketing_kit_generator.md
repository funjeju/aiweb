# 04_marketing_kit_generator.md

# 1. Marketing Kit Generator Overview (마케팅 키트 생성 시스템)

본 기능의 목적은 단순한 홍보물 제작이 아니다. 핵심은 **웹사이트를 만들면 운영에 필요한 마케팅 산출물이 자동으로 따라오는 구조**를 만드는 것이다.

즉, 본 시스템은 웹빌더의 부가기능이 아니라 웹빌더 가치 자체를 증폭시키는 기능이다.

사용자는 웹사이트를 만든 뒤 별도의 디자인 작업 없이 다음과 같은 결과물을 즉시 생성할 수 있어야 한다.

* 디지털 명함
* 메뉴 포스터
* QR 메뉴판
* SNS 카드 이미지
* 이벤트 홍보 카드
* 프로모션 배너

핵심 철학은 다음과 같다.

> One Source → Multi Use

한 번 입력된 데이터가 여러 채널에서 재사용되어야 한다.

---

# 2. Core Philosophy (핵심 철학)

사용자는 디자인을 다시 하면 안 된다.

웹사이트를 구성하는 JSON 상태값(SiteSchema)이 모든 마케팅 산출물의 단일 데이터 소스(Single Source of Truth)가 된다.

즉, 사용자가 웹사이트에서 수정한 정보는 자동으로 명함·포스터·SNS 카드에도 반영된다.

예를 들어 웹사이트에서 메인 컬러를 따뜻한 오렌지 톤으로 바꾸면, 메뉴판 포스터와 명함의 포인트 컬러도 함께 변경된다. 메뉴명을 수정하면 QR 메뉴판과 SNS 카드 텍스트 역시 동기화된다.

사용자는 “다시 만드는 경험”이 아니라 **자동으로 이어지는 경험**을 받아야 한다.

---

# 3. One Source Multi Use Structure

마케팅 키트는 웹사이트 데이터를 그대로 재활용한다.

예를 들어 `merchantInfo`, `designTokens`, `contentAssets`, `menuData`, `layout` 데이터가 여러 산출물 생성의 재료가 된다.

예시 흐름:

메뉴 사진 업로드

→ OCR 분석

→ menuData 자동 생성

→ 웹사이트 메뉴 섹션 자동 생성

→ 메뉴판 포스터 생성

→ QR 메뉴판 생성

→ SNS 메뉴 카드 생성

즉,

> 한 번 입력하면 끝

이어야 한다.

---

# 4. Core Rendering Architecture

과거 방식처럼 Photoshop 수준의 무거운 그래픽 엔진을 쓰지 않는다.

본 시스템은 **React 컴포넌트를 그대로 이미지화(rendering)** 하는 방식을 사용한다.

권장 스택:

* `@vercel/og (Satori)`
* `html2canvas`
* `jspdf`

작동 방식은 단순하다.

웹사이트에서 사용하는 디자인 시스템과 React 컴포넌트를 그대로 재사용하여 이미지로 굽는다(bake).

즉, 명함·포스터·SNS 카드가 웹사이트와 다른 디자인 언어를 가지면 안 된다.

핵심 원칙은 다음과 같다.

> 웹사이트 = 브랜드 디자인 원본

이다.

---

# 5. Digital Business Card (디지털 명함)

디지털 명함은 가장 즉각적인 산출물이다.

사장님이나 대행자는 웹사이트 제작 후 곧바로 명함 이미지를 생성할 수 있어야 한다.

명함은 다음 데이터를 자동 사용한다.

* 상호명
* 전화번호
* 주소
* 웹사이트 URL
* 브랜드 컬러
* 대표 이미지
* 로고

특히 QR 코드가 핵심이다.

QR 스캔 시 해당 웹사이트로 이동해야 하며, 사용자는 별도의 링크 작업을 할 필요가 없어야 한다.

예시 UX:

웹사이트 생성 완료

→ [명함 만들기]

→ 자동 생성

→ PNG 저장

→ 카카오톡 공유 가능

명함 디자인 역시 블록 기반 variation 구조를 가진다.

예:

* Minimal Card
* Premium Card
* Warm Cafe Card
* Modern Business Card

AI는 업종과 분위기에 맞는 variation을 추천한다.

# 6. Menu Poster Generator (핵심 기능)

메뉴판 생성 기능은 매우 중요하다.

실제로 많은 소상공인은 웹사이트보다 **메뉴판 이미지 제작 니즈**가 더 크다.

따라서 메뉴판 생성은 단순 부가기능이 아니라 웹빌더 가치 증폭 장치로 본다.

가장 중요한 UX는 다음이다.

> 사진 한 장으로 메뉴판 생성

예시:

카페 메뉴판 사진 업로드

→ OCR 분석

→ 메뉴명/가격 자동 추출

→ 정돈된 메뉴판 디자인 생성

→ 웹사이트 메뉴 섹션 동기화

→ SNS 메뉴 카드 생성

→ QR 메뉴판 생성

사용자는 메뉴를 다시 타이핑하지 않아야 한다.

---

# 7. Menu OCR Flow

메뉴 OCR은 다음 순서를 따른다.

사용자 입력:

메뉴 사진 업로드

AI 처리:

1. OCR 수행
2. 메뉴명 추출
3. 가격 추출
4. 설명 자동 생성(Optional)
5. 카테고리 분류(Optional)

예:

아메리카노 5,000
카페라떼 6,000

↓

```json
{
  "menuItems": [
    {
      "name": "아메리카노",
      "price": 5000
    },
    {
      "name": "카페라떼",
      "price": 6000
    }
  ]
}
```

↓

자동 생성:

* Website Menu Block
* Menu Poster
* QR Menu
* SNS Menu Card

핵심은 다음이다.

> 입력은 한 번, 활용은 여러 번

---

# 8. Poster Template System

포스터는 자유 디자인 방식이 아니다.

검증된 템플릿 조합을 제공한다.

예:

### Type A — Signature Menu Poster

대표 메뉴 사진 중심

카페/식당 추천

### Type B — Text Menu Poster

텍스트 위주

가격표 중심

### Type C — Review/Event Poster

후기·이벤트 중심

프로모션용

### Type D — Promotion Card

SNS 업로드 최적화

세일/행사/신메뉴 소개

핵심 원칙은 다음이다.

> Create(창작)

가 아니라

> Curate(큐레이션)

이다.

---

# 9. SNS Card Generator

웹사이트 데이터를 기반으로 SNS 카드도 자동 생성한다.

예:

신메뉴 등록

↓

자동 카드 생성

↓

인스타 업로드 가능

예시 결과물:

* 정사각 카드 (Instagram)
* 세로형 카드 (Story)
* 이벤트 배너
* 메뉴 소개 카드

핵심은:

> 콘텐츠를 다시 만들지 않게 한다

이다.

# 10. Dashboard UX

관리자 화면에서 접근은 매우 쉬워야 한다.

위치:

`brand-kit`

구조:

상단 탭:

* 명함
* 메뉴판
* SNS 카드
* 이벤트 포스터

중앙:

실시간 미리보기

하단:

* PNG 다운로드
* PDF 다운로드
* 공유하기

사용자는 디자인 툴을 배울 필요가 없어야 한다.

---

# 11. Agency Workflow

대행자는 반복 생산성이 중요하다.

예시:

플레이스 복붙

→ 사이트 생성

→ 메뉴 사진 업로드

→ 메뉴판 자동 생성

→ 명함 생성

→ 링크 전달

목표:

> 5~10분 안에 운영 패키지 완성

즉, 웹사이트 하나를 만들면 실제 영업에 필요한 기본 홍보물이 함께 나오는 구조다.

---

# 12. Final Principle

마케팅 키트는 독립 제품이 아니다.

본질은:

> 웹빌더 가치 증폭 시스템

이다.

웹사이트를 만들면 자연스럽게 운영 툴과 마케팅 자산이 따라와야 한다.

핵심은 결국 다음 한 문장이다.

> 한 번 만들면, 여러 곳에서 자동 활용된다.
