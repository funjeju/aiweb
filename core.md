# Project Core: AI Local Business Website Builder

## (초간편 소상공인 웹빌더 + AI 운영 자동화 플랫폼)

---

# 1. Project Overview (프로젝트 개요)

본 프로젝트는 소상공인 및 로컬 비즈니스를 위한 **“가장 쉽게 웹사이트를 만들 수 있는 AI 웹빌더”**이다.

핵심 목표는 복잡한 홈페이지 제작 과정을 제거하고, 누구나 **3분 안에 실사용 가능한 가게 웹사이트를 만들 수 있게 하는 것**이다.

사용자는 네이버 플레이스 URL, SNS 링크 또는 최소한의 정보 입력만으로 웹사이트 초안을 생성할 수 있으며, AI는 이를 자동으로 정리하여 모바일 최적화된 반응형 웹사이트를 즉시 생성한다.

그러나 본 서비스의 본질은 “AI 생성기”가 아니다.

**웹빌더(Web Builder)가 제품의 핵심(Core)**이며, AI는 사용자의 입력 부담을 줄이고 생산성을 극대화하는 **가속기(Accelerator)** 역할을 수행한다.

즉,

> “AI가 만들어주는 웹사이트”가 아니라
> “누구나 쉽게 운영할 수 있는 웹빌더를 AI가 더 쉽게 만들어주는 구조”

를 지향한다.

---

# 2. Core Value (핵심 가치)

## 2.1 가장 쉬운 웹사이트 제작 경험

기존 웹빌더는 설정 메뉴, 디자인 옵션, 배치 수정 등이 복잡하다.

본 서비스는 다음 원칙을 따른다.

* 입력 최소화
* 즉시 결과 제공
* 수정 난이도 최소화
* 디자인 실패 방지

사용자는 URL 붙여넣기 또는 몇 가지 정보 입력만으로 사이트 초안을 즉시 얻는다.

예시:

* 네이버 플레이스 URL 입력
* 인스타그램 링크 입력
* 메뉴판 사진 업로드
* 대표 사진 업로드

→ AI가 자동으로 사이트 생성

---

## 2.2 Zero-to-Low Input 전략

본 서비스는 완전 자동(Zero Input)만을 목표로 하지 않는다.

현실적인 성공률을 위해 **Low Input(Fallback) 구조**를 함께 지원한다.

### Mode A. Zero Input

입력:

* 네이버 플레이스 URL
* 인스타그램 URL

결과:

* 상호 정보 자동 수집
* 카피 자동 생성
* 메뉴/섹션 자동 구성
* 웹사이트 초안 생성

### Mode B. Low Input (Fallback)

스크래핑이 부족하거나 대행 제작 상황에서는 사용자가 최소 정보만 입력한다.

예:

* 상호명
* 업종 선택
* 대표 사진
* 메뉴 사진
* 전화번호
* 소개 문구

AI는 부족한 부분을 보완하여 사이트를 완성한다.

즉,

> 자동 생성 실패 = 서비스 실패

가 아니라

> 최소 입력으로 즉시 복구

가 설계 원칙이다.

---

## 2.3 대행 제작(Agency) 친화적 구조

본 서비스는 소상공인 직접 사용뿐 아니라 **대행 제작자(Agency Operator)** 시나리오를 강하게 고려한다.

대행자는 여러 개의 매장을 빠르게 제작해야 하므로 생산성이 중요하다.

대행 흐름 예시:

네이버 플레이스 정보 복사 → 메뉴 사진 업로드 → 사이트 자동 생성 → 텍스트 수정 → 즉시 퍼블리싱

이를 통해 1개 사이트 제작 시간을 극단적으로 단축한다.

즉, 본 서비스는 단순 SaaS를 넘어 **초저가 웹사이트 제작 대행 툴**로도 활용 가능하다.

# 3. Product Philosophy (제품 철학)

## 3.1 웹빌더가 본체, AI는 가속기

제품 우선순위는 다음과 같다.

1. 웹빌더 기능 완성도
2. AI 자동화
3. 실시간 데이터 파이프라인(DaaS)

AI는 사용자가 웹사이트를 “쉽게 만들도록” 돕는 도구일 뿐이며, 제품의 중심은 항상 웹빌더에 있다.

---

## 3.2 보이는 그대로 편집 (WYSIWYG)

웹사이트 화면 자체가 편집기다.

복잡한 관리자 페이지를 제거한다.

사용자는 다음 행동만으로 수정 가능해야 한다.

* 클릭 → 텍스트 수정
* 드래그 → 순서 변경
* 버튼 클릭 → 디자인 교체
* 사진 추가 → 즉시 반영

“디자인은 예쁘게 고정하고 콘텐츠만 쉽게 수정”하는 것이 핵심 철학이다.

---

## 3.3 실패하지 않는 디자인

웹사이트는 자유도가 높을수록 오히려 망가진다.

따라서 본 서비스는 **제한된 블록 기반 웹빌더** 구조를 채택한다.

예:

* Hero
* Featured Menu
* Gallery
* Review
* CTA
* Map
* Contact
* Live Feed

AI는 자유 생성 대신 **검증된 조합을 큐레이션**한다.

---

# 4. AI-Assisted Website Generation (AI 기반 생성 구조)

AI는 웹사이트를 “창작”하지 않는다.

정확히는 다음을 수행한다.

### 1단계 — 정보 구조화

비정형 데이터를 구조화한다.

예:

* 플레이스 정보
* 리뷰
* 메뉴
* 이미지
* 소개글

↓

정형 JSON 변환

### 2단계 — 템플릿 큐레이션

업종과 분위기에 따라 가장 적절한 템플릿 조합을 추천한다.

예:

카페형:

Hero + Featured Menu + Gallery + Review + Map

펜션형:

Hero + Room Gallery + Reservation + Review + Map

미용실형:

Hero + Style Gallery + Price List + Booking

### 3단계 — 카피라이팅 보조

리뷰와 소개 문구를 바탕으로 마케팅 카피를 자동 생성한다.

---

# 5. Pattern Library + Reference System

본 서비스는 AI가 랜덤하게 페이지를 생성하지 않는다.

AI 품질 안정화를 위해 **Pattern Library 기반 설계**를 채택한다.

구조:

### Design System

고정 규칙

* spacing
* radius
* color system
* typography
* layout rule

### Pattern Library

검증된 UI 블록 조합

예:

HeroCentered v1
HeroCentered v2
FeatureSplit v3
MenuGrid v2

### Reference Library

업종 및 감성 레퍼런스 이미지

예:

* 제주 감성 카페
* 모던 미용실
* 감성 펜션
* 미니멀 식당

AI는 이미지 자체를 복제하지 않고, **분위기와 스타일 힌트**로만 활용한다.

즉,

> 창작(Create)

이 아니라

> 큐레이션(Curate)

방식이다.

---

# 6. Marketing Automation (운영 자동화)

웹사이트는 시작점일 뿐이다.

한 번 입력된 데이터는 여러 산출물로 재활용된다.

예:

* 디지털 명함
* 메뉴 포스터
* SNS 카드
* QR 메뉴판
* 프로모션 이미지

예시:

메뉴 사진 업로드

↓

AI OCR

↓

메뉴명 + 가격 추출

↓

웹사이트 메뉴 자동 생성

↓

메뉴판 포스터 자동 생성

---

# 7. Future Layer: DaaS (백엔드 데이터 레이어)

DaaS(Data as a Service)는 사용자에게 전면 노출되는 핵심 기능이 아니다.

이는 서비스 내부에서 동작하는 후면 엔진(Backend Layer)이다.

EXIF 기반 실시간 데이터와 로컬 정보를 축적하여 향후 관광 플랫폼, 컨시어지, 추천 엔진 등과 연동 가능한 구조를 만든다.

그러나 제품 관점에서의 우선순위는 항상 다음과 같다.

> 웹빌더 → 운영 자동화 → 데이터 확장

---

# 8. Final Product Definition

본 서비스는 다음과 같이 정의된다.

> “소상공인과 대행자가 가장 쉽게 웹사이트를 만들고 운영할 수 있게 하는 AI 웹빌더”

핵심은 AI가 아니다.

핵심은

> 가장 쉬운 웹사이트 제작 경험

이다.
